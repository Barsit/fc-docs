# 内置模板

内置模板由云沙箱平台提供和维护，用于快速创建标准 Sandbox。当前可直接使用的内置模板覆盖基础命令与文件访问、代码执行场景。如果您还需要浏览器自动化或代码执行与浏览器协同的复合场景，可以通过预构建镜像自行构建对应模板，具体流程见下方模板列表。

如果您只是需要 Python、Node.js、文件管理或代码执行等通用能力，建议优先使用内置模板。只有当内置模板无法满足业务依赖、系统库、运行时版本或企业标准化要求时，再创建自定义镜像模板。

## 模板列表

| 模板 | 适用场景 | 说明 |
| --- | --- | --- |
| [base 模板](02.内置模板/base-模板.md) | 基础命令、文件访问、SDK 连通性验证 | 默认基础模板，不提供 Code Interpreter 服务或浏览器自动化能力 |
| [code-interpreter-v1 模板](02.内置模板/code-interpreter-v1-模板.md) | AI Agent 代码执行、数据分析、文件处理 | 快速入门和代码解释器示例默认使用 `code-interpreter-v1` |
| [browser 模板](02.内置模板/browser-模板.md) | 浏览器自动化、网页抓取、截图、UI 测试 | 提供 CDP、VNC、截图、页面交互等浏览器能力，需通过预置镜像构建模板后使用 |
| [All-In-One 模板](02.内置模板/all-in-one-模板.md) | 代码执行和浏览器自动化协同 | 同时提供 Code Interpreter 能力和 browser 能力，需通过预置镜像构建模板后使用 |

> **说明**
>
> `base` 与 `code-interpreter-v1` 在账号开通云沙箱后会自动就绪，可直接用模板名创建沙箱。`browser` 与 `All-In-One` 提供预置镜像，需要先使用对应镜像构建一个带名称的模板，再用该模板名创建沙箱，具体流程见各自的模板文档。

## 查看模板

使用 CLI 查看当前账号可用模板：

```bash
export E2B_API_KEY="<your-api-key>"
export E2B_API_URL="https://api.cn-beijing.e2b.fc.aliyuncs.com"
export E2B_DOMAIN="cn-beijing.e2b.fc.aliyuncs.com"

e2b template list
```

## 使用模板创建 Sandbox

使用 `e2b_code_interpreter` SDK 时，默认创建 `code-interpreter-v1` 沙箱，通常不需要显式指定模板。

使用通用 `e2b` SDK 时，可通过 `template` 参数显式选择模板。Python 示例：

```python
import os

from e2b import Sandbox

sandbox = Sandbox.create(
    template="code-interpreter-v1",
    api_key=os.environ["E2B_API_KEY"],
    api_url=os.environ["E2B_API_URL"],
    domain=os.environ["E2B_DOMAIN"],
)
```

TypeScript 示例：

```typescript
import { Sandbox } from "e2b";

const sandbox = await Sandbox.create("code-interpreter-v1", {
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
});
```
