# 管理模板

模板管理包括查看模板、创建模板、跟踪构建状态，以及使用标签和别名组织模板版本。

## 查看模板列表

使用 E2B CLI：

```bash
e2b template list
```

也可以通过 E2B SDK 检查模板是否存在：

```python
import os
from e2b import Template

# 统一连接配置，后续调用复用
opts = {
    "api_key": os.environ["E2B_API_KEY"],
    "api_url": os.environ["E2B_API_URL"],
    "domain": os.environ["E2B_DOMAIN"],
}

# 检查模板名称是否存在
Template.exists("my-template", **opts)

# 检查别名是否存在
Template.alias_exists("my-alias", **opts)
```

具体接口边界参见[E2B 兼容说明](../../01.开始使用/05.E2B%20兼容说明.md)。

## 创建模板

使用 `Template.build` 创建自定义模板。详见[构建自定义镜像模板](03.构建自定义镜像模板.md)。

## 构建状态和日志

构建模板时，应保留构建 ID 和模板 ID，便于排查构建失败、镜像拉取失败或依赖安装失败等问题。

使用 `build_in_background` 异步构建并轮询状态：

```python
import os
import time
from e2b import Template, default_build_logger

opts = {
    "api_key": os.environ["E2B_API_KEY"],
    "api_url": os.environ["E2B_API_URL"],
    "domain": os.environ["E2B_DOMAIN"],
}

build = Template.build_in_background(
    Template().from_image(os.environ["FROM_IMAGE"]),
    name="my-template",
    cpu_count=2,
    memory_mb=2048,
    on_build_logs=default_build_logger(),
    **opts,
)

print(f"template_id: {build.template_id}")
print(f"build_id: {build.build_id}")

while True:
    status = Template.get_build_status(build, logs_offset=0, **opts)
    print(f"build_status: {status.status.value}")
    if status.status.value == "ready":
        break
    if status.status.value == "error":
        # error 时 reason 为 BuildStatusReason 对象，ready/building 时为 None
        reason = status.reason.message if status.reason else "unknown"
        raise RuntimeError(f"build failed: {reason}")
    time.sleep(5)
```

构建状态取值：`waiting`、`building`、`ready`、`error`。`error` 时 `status.reason` 是含失败原因的对象，可从 `status.reason.message` 获取错误消息，配合 `log_entries` 查看日志条目。

## 标签和别名

标签和别名适合区分模板版本、业务场景和灰度阶段。生产环境建议先创建新模板并验证，再切换业务使用的模板别名。
