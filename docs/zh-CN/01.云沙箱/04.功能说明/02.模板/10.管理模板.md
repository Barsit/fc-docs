# 管理模板

模板管理包括查看模板、创建模板、跟踪构建状态，以及使用标签和别名组织模板版本。

## 查看模板列表

使用 E2B CLI：

```bash
e2b template list
```

也可以通过 E2B SDK 检查模板是否存在：

```python
import os
from e2b import Template

# 统一连接配置，后续调用复用
opts = {
    "api_key": os.environ["E2B_API_KEY"],
    "api_url": os.environ["E2B_API_URL"],
    "domain": os.environ["E2B_DOMAIN"],
}

# 检查模板名称是否存在
Template.exists("my-template", **opts)

# 检查别名是否存在
Template.alias_exists("my-alias", **opts)
```

具体接口边界参见[E2B 兼容说明](../../01.开始使用/05.E2B%20兼容说明.md)。

## 创建模板

使用 `Template.build` 创建自定义模板。详见[构建自定义镜像模板](03.构建自定义镜像模板.md)。

## 构建状态和日志

`Template.build` 同步构建会阻塞至完成；如需异步构建，使用 `Template.build_in_background`，并通过 `Template.get_build_status` 轮询构建状态。状态取值：`waiting`、`building`、`ready`、`error`。构建失败（`error`）时，从返回值的 `reason.message` 获取失败阶段和原因。构建失败排查见[错误处理](11.错误处理.md)。

## 标签和别名

标签和别名适合区分模板版本、业务场景和灰度阶段。生产环境建议先创建新模板并验证，再切换业务使用的模板别名。
